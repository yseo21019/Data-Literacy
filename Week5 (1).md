# 1️⃣ 개념정리

## 01. 자꾸만 늘어가는 '방법맨'이란

```
✅ 학습 목표 :
* '방법맨'이란 무엇인지 설명할 수 있다.
* 문제 해결보다 기법 자체에 집착하게 되는 사고 방식의 위험성을 설명할 수 있다. 
```

### 문제 해결 프로세스에서 누구나 하는 실수
**곧바로 해결방안에 무턱대고 달려드는 것!**
- 해결방안이 참신한 아이디어만 내는거면, 애당초 데이터는 필요 없음
- Ex) 인구가 감소하고 있다 -> 이벤트와 투어를 많이 실시하면 인구 감소를 완화할 수 있다
  - 데이터를 통해 확인한 내용(인구가 감소하고 있다)과 제안 내용에 직접적인 관계가 없다

### 결과와 유효성에 차이가 생기는 '원인' 파악 유무
**좋은 해결방안을 생각해내기 위해서는 원인을 분석해야함!**
- Ex) '시간 외 근무가 많다'라는 문제의 해결방안 사례
  - **1단계**: 수요일이 유독 시간 외 근무가 많다 -> 수요일을 정시 퇴근일로 지정
  - **2단계**: H 부서가 유독 다른 부서에 비해 시간 외 근무가 많다 -> H부 부장에게 보고
  - **3단계**: H부서의 문제점을 분석해보니 효율성의 문제(역량 부족, 사기 문제), 업무량의 문제 (부적절한 업무분담, 돌발 업무) 등이 있다 -> 업무 수행 관련 교육 실시, 평가 기준 개선 등 **근본적인 해결책에 가까워질 수 있음**


## 02. 방법맨이 돼버리는 구조

```
✅ 학습 목표 :
* 방법맨이 되어버리는 구조적 요인을 이해한다.
* 분석 기법보다 '문제 인식 -> 사고 구조 설계'가 우선임을 인식한다. 
```
                        
### '생각한다'의 두가지 의미
- 풍부한 창의성으로 혁신적인 아이디어 생각해내기
- 논리적으로 꾸준히 사물이나 현상의 원리 추구하기

-> 무조건적으로 참신한 아이디어를 생각해내는 것이 좋은 것은 아님.
 
 목적과 문제, 원인을 확실하게 하고 해결 방안을 생각해내는 과정이 필수적임.





## 03. '왜(원인)' 그런지 끈질기게 생각하라

```
✅ 학습 목표 :
* 기법을 적용하기 전에 '무엇을, 왜 분석하는가'를 지속적으로 질문하는 태도를 기른다.
* 단순 결과에 안주하지 않고 원인을 집요하게 파고드는 분석 사고력을 기른다. 
```


- 지금 내가 보고 있는 것이 문제인지, 원인인지, 해결방안인지 구분하기
   - 문제 (ex. 매출이 감소했다)
   - 원인 (ex. 방문객 수 감소 / 경쟁사 가격 인하)
   - 해결방안 (ex. 이벤트 진행, 가격 조정 등)
- 원인은 단순하지 않다. ‘왜?’를 여러 번 반복하며 깊게 파고들어야 한다.
   - Ex) 매출 감소 → 왜? 방문객 감소 → 왜? 홈페이지 업데이트 빈도 감소 → 왜? 담당자 인력 축소 …
이런 식으로 **근본 원인(Bottom cause)** 에 도달해야 진짜 해결방안이 나온다.
- 해결방안을 고민하는 것은 문제 해결의 ‘마지막 단계’
   - 원인이 명확히 규명되기 전에 해결방안을 먼저 생각하면
→ 효과 없는 임시방편만 나오고 **‘방법맨’** 이 된다.
