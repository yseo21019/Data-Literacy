# 1️⃣ 개념정리

## 01. 데이터 활용에 실패하는 두 가지 이유!

```
✅ 학습 목표 :
* 데이터 분석이 실무에서 실패하는 일반적인 원인을 이해한다.
* 문제 정의와 데이터 선택의 중요성을 인식하고, 실패를 유발하는 사고 패턴을 설명할 수 있다. 
```

* 대부분 데이터 활용을  못하는 이유는 방법론의 문제라고 생각한다. 그러나 실제 이유는 다음과 같다.
1. 풀고자 하는 문제가 명확하지 않다.
2. 정의한 문제와 사용하는 데이터가 일치하지 않는다.

* 극복방법: 데이터에서 무엇을 획득할 수 있을지 우선적으로 생각하고, 그 후에 데이터를 가공하기. 



## 02. 주요 원인 1 : 풀고자 하는 문제가 명확하지 않다.

```
✅ 학습 목표 :
* 문제 정의의 불명확성이 분석 오류로 이어지는 과정을 설명할 수 있다.
* '문제','원인','해결'의 구분을 명확히 하고, 목적 지향적 사고를 적용할 수 있다. 
```

주어진 데이터를 가지고 무엇을 해야할지, 어떤 문제를 정의해야할지는 정해져있지 않다. 그러나 자신이 데이터를 분석하는 목적과, 해결할 문제를 정의할 필요성은 있다. 

목적과 문제를 정의할 때 확인해야할 포인트
1. 사용된 언어가 구체적이고 명확한가? 
2. '문제', '원인', '해결방안'을 구분하고 있는가?



## 03. 주요 원인 2 : 정의한 문제와 사용하는 데이터가 일치하지 않는다

```
✅ 학습 목표 :
* 분석에 사용하는 데이터가 정의한 문제를 반영하고 있는지 판단할 수 있다.
* 데이터와 문제의 불일치 사례(지표 불일치 등)를 통해 실무에서의 데이터 활용 한계를 인식한다. 
```

짐작이나 추측이 아닌 논리적인 설명을 통해 지표를 결정해야한다.

결론 도출 프로세스
 - 평가 지표 및 기준을 결정
 - 적합한 데이터 및 그래프 선택
 - 결론



## 04. 비즈니스 현장 사례 분석

~~~
✅ 학습 목표 :
* 실제 현장에서 발생하는 데이터 분석 문제를 사례를 통해 설명할 수 있다.
* 올바른 데이터 활용하는 방법을 인식한다. 
~~~

지표를 통해 현재 상황을 파악하고, 직면한 문제를 어떻게 나타낼지에 대한 관점으로 지표를 고려할 필요가 있다.
