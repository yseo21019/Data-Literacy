# 1️⃣ 개념정리

## 01. 결과와 결론은 다르다

```
✅ 학습 목표 :
* 데이터 분석 결과와 해석을 통해 내리는 결론을 구분되어야 함을 이해한다.
```

### 결과와 결론의 차이: 목적을 이해하지 못하는 사례
- 결과: 계산과 분석을 해서 나온 결과물
  - 메일링 리스트 등록 여부를 통해 비교할 경우, 등록한 사람의 방문 횟수가 더 높음.
  - 여성 고객의 경우, 등록한 사람이 그렇지 않은 사람보다 평균값 높음.
  - 남성 고객의 경우, 메일 수신 여부와 관계 없이 방문 횟수 평균값 비슷.
- 결론: 그 결과가 목적에 대해 어떤 의미가 있는지 설명하는 것
  - 메일링 리스트 등록은 방문 횟수 증가에 효과적임.
  - 메일링 리스트 등록은 여성 고객에게 효과적이지만, 남성 고객에게는 그렇지 않음.
> 메일링 서비스 효과를 확인하고자하는 **목적**과 관련된 내용을 설명하고 있음 -> 이것이 '결론'


## 02. '결과'와 '결론'의 차이 : 데이터를 너무 단순하게 이해한 사례 

```
✅ 학습 목표 :
* 데이터 수치를 지나치게 단순하게 해석할 때 발생할 수 있는 오류를 사례를 통해 파악한다. 
* 동일한 결과라도 결론은 다양하게 나올 수 있음을 인식하고, 관점의 중요성을 설명할 수 있다. 
```

### 집세 비교 사례
 - 평균만 보면 교외 집세가 더 저렴해 보임.
 - 그러나 표준편차와 범위를 보면

  → 도심이 가격 선택 폭이 더 넓음 (좋은 매물 발견 가능성 ↑).
 - 결론: 값이 싸다는 ‘평균’보다, 선택 가능한 범위(폭) 가 더 중요하다. “교외가 싸지만, 좋은 조건을 원한다면 도심부터 보는 게 유리하다.”

### 초콜릿 판매 데이터 사례
- 데이터 자체에는 정답이 없음 → 먼저 ‘무엇을 말하고 싶은가’(목적) 를 정의해야 함.
- 그래프는 단순히 보여주는 것이 아니라 → 결론을 설득하는 도구로 사용해야 함.
- 예시 결론:
판매량이 가장 많은 E 초콜릿에 집중하고,
특히 많이 팔리는 나고야 지점에 자원을 우선 투자.
- 또 다른 결론 예시:
판매량이 낮은 A, D 제품을 나고야·오사카 지점 중심으로 개선.
> 그래프 자체가 결론이 되는 것이 아닌, 그래프를 통해 결론을 설득시키자!


## 03. 결론을 낼 때 주의해야 할 점

```
✅ 학습 목표 :
* 결론 도출 시 반드시 점검해야할 부분을 습득한다. 
* 데이터를 통해 얻은 인사이트를 왜곡 없이 요약하는 방법을 설명할 수 있다. 
```
### 데이터로 알 수 있는 범위를 벗어나지 말 것
- 결론은 데이터가 말해주는 선에서만 내려야 함.
- 데이터에 없는 이유나 배경을 상상해서 붙이면 안 됨.
- 즉, ‘추측’과 ‘사실’을 섞지 말 것.

### 자신의 해석이 개입되는 순간 오류 발생
- 사람은 본능적으로 자신에게 유리한 방향으로 해석하고 싶어함 → 인지 편향.
- 결론을 낼 때 “이건 데이터가 말하는가? 아니면 내가 그렇게 믿고 싶은가?” 를 구분해야 함.

### 정답은 한 가지가 아니다
- 데이터는 단서일 뿐,여러 개의 결론이 모두 가능할 수 있음.
- 중요한 것은 **내가 내린 결론을 데이터로 설명할 수 있는가**임.

## 문제

> **🧚Q. 맛집을 탐방하여 이를 인스타를 활용하여 홍보하는 미정이는 SNS 콘텐츠 개선을 위해 인스타그램 게시물 통계를 분석했다. 지난 3개월간 업로드된 게시물 30개의 좋아요 수 평균이 120개였고, 그중에서 가장 많은 좋아요(480개)를 받은 게시물이 '중앙대학교 특집 맛집 탐방'이라는 결과를 도출했다.**
>
> **이를 보고 미정이는 이러한 생각을 했다. '중앙대 근처 맛집 탐방 콘텐츠가 압도적으로 반응이 좋으니, 앞으로는 대학가 근처의 콘텐츠를 제작하는게 맞는 것 같다.'**
>
> **미정이의 분석 결과와 결론 사이에는 어떤 논리적 비약 또는 해석의 오류가 있나요? 아래 항목을 중심으로 2~3가지 근거를 들어 설명해보세요.**
>
> **'좋아요 수'라는 결과 지표 / 하나의 사례만으로 결론 도출 / 다른 가능성 검토 여부**
